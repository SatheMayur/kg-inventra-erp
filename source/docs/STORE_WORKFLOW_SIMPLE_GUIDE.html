KG Store Simple Workflow Guide

One-page guide for request, purchase order, receiving, and issue flow

Main Flow

  1. Employee creates request: Add item, quantity, department, purpose, and priority.
  2. Department head approves: Request moves forward only after approval.
  3. Store checks stock: If stock is available, it is reserved for the request.
  4. Shortage goes to PO: Purchase order is created only for shortage quantity.
  5. Store receives PO items: Received stock is added and reserved back to the original request.
  6. Store issues material: Ready quantity is issued to the employee and request is closed when complete.

What Each Status Means

  • PendingWaiting for approval.
  • ApprovedStore can issue if stock is ready.
  • Converted to POPurchase order created for shortage.
  • Ready for IssueStock is reserved and can be issued.
  • IssuedMaterial has been given to employee.

Who Does What

  • Employee: Create request and track status.
  • Department Head: Approve or reject request.
  • Store Admin / Operator: Check stock, receive goods, issue items.
  • Purchase User: Create and follow up purchase orders.
  • Accounts: Check invoice and GST totals.

Next Action Guide

  • Approve Request: Department head must approve first.
  • Create PO: Stock shortage exists and purchase is required.
  • Receive PO Stock: PO is created, waiting for items to arrive.
  • Proceed to Issue: Stock is reserved and ready for employee.
  • Completed: All approved items are issued.

Important Rules

  • Do not issue more than reserved and ready quantity.
  • Create PO only for shortage quantity, not full requested quantity.
  • After PO receipt, check the original request for ready-to-issue status.
  • Add transportation cost, CGST, SGST, and IGST in PO before final save.
  • Use invoice matching before closing the purchase cycle.
